# 2026-06-24 记忆

## 视频分析交互Workflow · B站支持 + 分析质量修复

### B站视频支持 ✅
- **根因**: yt-dlp 不识别 b23.tv 短链，且 B站 API 境外 IP 封锁
- **修复**: 
  - 后端 `server.py` 新增 you-get 下载器（B站专用，境外可下载）
  - 前端 `index.html` 新增 `resolveB23Url()` 解析 b23.tv 短链（利用用户浏览器国内IP）
- **文档**: `video_analyzer_app/VIDEO_DOWNLOAD_STRATEGY.md`

### 视频截断修复 ✅
- **根因**: ffmpeg `-t 180` 硬编码截断长视频，导致脚本解析只到 02:59
- **Daryl批评**: 只盯着删参数，没考虑全管道前后配合。不要为"省空间"牺牲"分析质量"
- **修复**: 简化策略——压缩但不截断，仅 >50MB 触发关键帧（标注降级）
- **原则**: 先对齐阶段目标（叙事正确），再搞分级复杂度（镜头/BGM 是 Phase 2）

### 脚本分析质量修复 ✅
- **根因**: `SCRIPT_ANALYSIS_PROMPT` 让 DeepSeek 直接接受 Gemini 解读，没有推理验证
- **Daryl指出错误**: 
  - "父亲挨揍"→应该是"儿子挨揍"（角色关系核实失败）
  - "忘作业"→应该是"故意留家里"（动机深度不够）
- **修复**: prompt 加入三步强制验证（角色关系、动机深度、逻辑连贯性）

### 镜头&声音分析优化 ✅
- **Daryl指出**: 情景剧不该全维度均等分析，要详略得当。剪辑节奏对留存率贡献大
- **修复**: `CINEMATOGRAPHY_PROMPT` 加入类型感知——先判断内容类型→识别有分析价值的维度→深挖重点/一笔带过
- **输出新字段**: `_contentType`, `_visualComplexity`, `_deepDims`, `_shallowDims`
