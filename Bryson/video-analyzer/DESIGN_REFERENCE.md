## 🏗️ 当前架构

### 前端
```
┌──────┬─────────────────────────────────────────────┐
│ 🖥️   │  顶部栏：链接输入 + 分析链接 + Demo 分析      │
│      ├──────────────┬──────────────────────────────┤
│ 📚   │ 视频播放器    │  分析面板（Tab 切换）:           │
│  素   │              │  📊 内容总览                    │
│  材   │ 对白卡片列表  │  🎬 脚本解析 (acts+张力+技巧)    │
│  库   │              │  📋 复刻模板 + 可复用元素        │
│      │ · 播放高亮    │  🎥 镜头&声音 (4维度推断)        │
│      │ · 点击跳视频  │  ⭐ 三项评分                    │
│      │ · 悬停标记    │  🏷️ 标记管理                    │
│      │ · ✕ 删除标记  │                                │
└──────┴──────────────┴──────────────────────────────┘
```

### 分析管线
```
yt-dlp(30s超时) → Whisper STT(本地免费) 
    → DeepSeek V4 Pro × 5 并行:
        ├── 内容总览 (overview)
        ├── 脚本解析 (script_analysis)
        ├── 复刻模板 (template + reusableElements)
        ├── 镜头&声音 (cinematography)
        └── 三项评分 (scoring)
    → ~$0.0025/次
```

### 技术栈
- 前端: Vanilla HTML/CSS/JS (单文件 ~1200行)
- 服务端: Python3 stdlib HTTP server (无外部依赖)
- AI: DeepSeek V4 Pro (chat API)
- STT: Whisper (本地 tiny 模型)
- 下载: yt-dlp + ffmpeg
- 公网: ngrok free tier → 端口 8777

## 🔑 待突破：Vision API

当前镜头&声音分析是基于对白文本推断，精度远不如 MVP 的逐帧分析。

### 需要的技术：
```python
# 帧提取
ffmpeg -i video.mp4 -vf "select='gt(scene,0.08)'" -vsync vfr keyframes/%04d.jpg

# Vision 分析（任选其一）
- DeepSeek VL2 (需确认 API 可用性)
- Gemini 2.5 Flash Vision (当前 403，需注册)
- OpenAI GPT-4o Vision (需有效 key)
```

### Vision 管线设计：
```
视频 → ffmpeg scene detect → 关键帧(10-30张)
    → Vision API 逐帧分析（景别/构图/光线/色彩/人物/情绪）
    → 汇总为结构化报告
    → 前端镜头&声音 Tab 显示精确数据
```
