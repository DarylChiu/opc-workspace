# 知识传输方案成本对比分析

**生成时间**: 2026-06-08  
**调用模型**: openrouter/anthropic/claude-sonnet-4.5

---

## 测试场景
- **内容**: IFRS 16 租赁准则讨论
- **时长**: 5分30秒
- **页数**: 2页 PDF（手写笔记）
- **复杂度**: 中英文混合，会计术语，结构化内容

---

## 方案对比

### 方案1：PDF + 音频（当前模式）

#### 用户操作
1. 三星 Tab S8 边写边录音
2. 导出 PDF + M4A 音频文件
3. 发送给 Self

#### Self 处理流程
1. Whisper 转录音频（5分30秒）
2. PyMuPDF 提取 PDF 为高清图片（每页 300 DPI）
3. Vision Model（Sonnet 4.5）识别手写内容
4. 人工对应 PDF 页面与音频段落
5. 整合成结构化笔记

#### Token 成本（实测）
| 项目 | Token 消耗 | 成本估算 |
|------|-----------|---------|
| 音频转录（Whisper local） | 0 | $0（本地处理） |
| PDF 转图片（PyMuPDF） | 0 | $0（本地处理） |
| 图片识别（2页 2500×3534px） | ~6000 tokens/页 | $0.015/页 × 2 = **$0.03** |
| 文本整理（Sonnet 4.5） | ~2000 tokens | $0.01 |
| **总计** | **~14k tokens** | **$0.04** |

#### 优点
- ✅ **用户负担最低**：自然书写，无需特殊操作
- ✅ **成本最低**：音频、PDF 处理都在本地，只有图片识别用 API
- ✅ **设备轻便**：Tab S8 随时可用
- ✅ **手写自由**：草稿风格，不需要工整

#### 缺点
- ⚠️ 需要 Self 人工推断 PDF 与音频的对应关系
- ⚠️ 多页 PDF 时对应难度增加

---

### 方案2：录屏讲解（视频 = 画面 + 声音）

#### 用户操作
1. 三星 Tab S8 屏幕录制 + 手写讲解
2. 导出视频文件（MP4/MOV）
3. 发送给 Self

#### Self 处理流程
1. ffmpeg 提取音频轨 → Whisper 转录
2. ffmpeg 按帧率提取关键帧（如每 5 秒 1 帧）
3. Vision Model 批量识别所有帧
4. 时间戳自动对应（帧号 = 时间轴）
5. 整合成结构化笔记

#### Token 成本（估算）

**视频参数假设**：
- 时长：5分30秒 = 330秒
- 分辨率：1920×1080（Tab S8 屏幕）
- 帧率：30fps
- 提取策略：每 5 秒提取 1 帧 → 共 66 帧

| 项目 | Token 消耗 | 成本估算 |
|------|-----------|---------|
| 音频转录（Whisper local） | 0 | $0 |
| 视频帧提取（ffmpeg） | 0 | $0（本地处理） |
| 图片识别（66帧 1920×1080px） | ~4000 tokens/帧 | $0.01/帧 × 66 = **$0.66** |
| 时间戳对齐与整理 | ~3000 tokens | $0.015 |
| **总计** | **~267k tokens** | **$0.675** |

#### 优点
- ✅ **画面与声音天然同步**：无需人工对应
- ✅ **能看到书写过程**：更直观理解思路
- ✅ **时间戳精确**：帧号 = 时间轴

#### 缺点
- ❌ **成本高 17 倍**：$0.675 vs $0.04
- ❌ **处理时间长**：66 帧逐一识别，约需 3-5 分钟
- ❌ **文件体积大**：5 分钟视频约 100-500MB（vs. PDF 571KB + 音频 5.1MB）
- ❌ **冗余信息多**：很多帧内容重复（手写中途、未完成状态）

### 方案3：PDF + 音频 + 口头标记（改进方案）

#### 用户操作
1. 三星 Tab S8 边写边录音
2. **翻页时口头说一句**："翻页" / "下一页" / "Page 2"
3. 导出 PDF + 音频，发送给 Self

#### Self 处理流程
1. Whisper 转录音频，识别 "翻页" 关键词
2. 提取时间戳（如 "翻页" 出现在 2:15、4:30）
3. PyMuPDF 提取 PDF 页面
4. 自动对应：第 1 页 = 0:00-2:15，第 2 页 = 2:15-4:30…
5. 整合成结构化笔记

#### Token 成本（估算）
| 项目 | Token 消耗 | 成本估算 |
|------|-----------|---------|
| 与方案1相同 | ~14k tokens | **$0.04** |

#### 优点
- ✅ **成本与方案1相同**：只增加一句口头标记
- ✅ **自动对应**：通过转录文本的时间戳精确定位
- ✅ **用户负担极低**：翻页时顺口说一句

#### 缺点
- ⚠️ 需要养成翻页标记的习惯（但非常轻量）

---

### 方案4：AI 自动推断对应（无需标记）

#### 技术原理
1. PDF 提取所有手写内容（关键词）
2. 音频转录提取所有口语内容（关键词）
3. 用 TF-IDF / 语义相似度匹配段落
4. 推断 "音频段落 A 对应 PDF 页面 X"

#### Token 成本（估算）
| 项目 | Token 消耗 | 成本估算 |
|------|-----------|---------|
| 基础识别（同方案1） | ~14k tokens | $0.04 |
| 语义匹配分析 | ~5000 tokens | $0.025 |
| **总计** | **~19k tokens** | **$0.065** |

#### 优点
- ✅ **用户零负担**：完全自动化
- ✅ **成本可控**：仅增加 $0.025（62.5% 增幅）

#### 缺点
- ⚠️ 准确度依赖内容关联性（如果 PDF 和音频内容差异大会误判）
- ⚠️ 需要开发自动匹配逻辑

## 推荐方案：方案3（PDF + 音频 + 口头标记）

### 理由
1. **成本最低**：$0.04/次，与当前模式相同
2. **用户负担极小**：翻页时说一句话（自然且不打断思路）
3. **准确度最高**：时间戳精确到秒
4. **可扩展性强**：适用于任意页数（10页、50页都 OK）

### 实施细节
- **口头标记词**：选一个自然的词，如"下一页"、"翻页"、"Page X"
- **识别逻辑**：Whisper 转录后用正则提取时间戳
  ```python
  import re
  transcript_with_timestamps = [
      {"start": 0.0, "text": "今天稍微讲一下 IFRS 16..."},
      {"start": 135.2, "text": "翻页"},  # ← 识别到标记
      {"start": 137.5, "text": "打击二房东出走的企业..."}
  ]
  ```
- **容错机制**：如果忘记说标记，Self 用方案4的 AI 推断兜底
