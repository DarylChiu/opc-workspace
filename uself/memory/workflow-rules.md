## 1. 决策授权矩阵

| 级别 | 响应时限 | 场景 | 处理方式 |
|------|---------|------|---------|
| **P0 紧急** | 30分钟 | 数据丢失、安全漏洞 | 执行应急方案，事后汇报 |
| **P1 重要** | 2小时 | 知识库架构、分类体系、核心结论 | 必须请示Daryl，提供2-3方案+推荐；超时→执行推荐方案 |
| **P2 正常** | 6小时 | 检索策略、索引优化、内容整理 | 可自主决策；超时→按最佳理解执行 |
| **P3 参考** | 24小时 | 文档完善、格式优化 | 不等待直接做 |

**核心原则**：知识整理归Agent，分类体系方向归Daryl
**超时机制**：绝不停滞，超时执行预设低风险方案
**禁止区**：私有知识外传、编造信息 → 绝对不可自主决策

## 3. 知识归纳 SOO（Standard Operating Procedure）
> Daryl 确认 2026-06-10

**触发条件**: 其他 Agent 通过 sessions_send 发送「知识归纳」消息给 Self

**四步流程**:
1. **归纳总结推理延伸**: 消化材料，提炼核心概念，做合理推理但不偏离主题
2. **知识树位置建议**: 给出加入哪个板块的建议方案 + 论据，等待 Daryl 确认
3. **建立索引**: Daryl 确认后，在 Index.md 中添加节点编号和路径
4. **缺口诊断**: 如知识点不够深入或全面，指出不足或补全（不偏题）

**汇报方式**: 飞书 DM 私聊给 Daryl，授权后执行归档到知识网络体系

## 5. 跨Agent知识接入流程 (Daryl确认 2026-06-10)

### 触发条件
收到其他 Agent 通过 `sessions_send` 发来的「知识归纳」类消息（包含讨论纪要、分析报告等需归档内容）

### 处理四步法
1. **归纳总结推理延伸**：提取核心命题 → 构建推理链 → 发现跨领域关联 → 标注置信度
2. **知识树位置建议**：分析内容属性 → 判断理论归属 → 给出放置位置 + 论据（含候选对比）
3. **建立索引页**：一级索引（根 README）+ 二级索引（子目录 README）+ 跨域索引
4. **缺口诊断**：指出内容不够深入或全面性不足的地方，提出补充建议（不偏题，不越界到原创内容）

### 汇报与授权
- 全部通过 DM 私聊向 Daryl 汇报，附完整方案
- Daryl 确认后执行归档
- 归档完成后通知内容提供方 Agent

### 约束
- 不替 Daryl 做分类决策，只给推荐方案
- 不确定的跨领域关联标注置信度
- 不为了补全面性而编造内容
