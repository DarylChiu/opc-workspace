## 一、核心发现摘要

### 🔑 最关键发现：双向时间轴联动

百万工作室的对白转写系统实现了 **「视频 ↔ 对白卡片」双向时间轴联动**：

| 方向 | 触发方式 | 效果 |
|------|---------|------|
| **视频 → 对白** | 视频播放中 | 当前播放段对应的对白卡片**自动蓝色高亮** |
| **对白 → 视频** | 鼠标悬停/点击卡片 | 显示「可以直接定位到对应的时间线」→ 视频跳转到该时间戳 |

这**正是 Daryl 想要的交互模式**：选取对白的卡片 → 标记对应视频位置。

## 三、对白转写卡片系统（重点分析）

### 3.1 卡片结构

位于「脚本详细拆解」页面的左栏，标题为 **「原文时间轴」**：

```
┌─────────────────────────────────────┐
│ 原文时间轴                     67条  │
│                                     │
│ ┌─────────────────────────────────┐ │
│ │ 00:25  方法                    │ │  ← 蓝色高亮 = 当前播放位置
│ │ 同时也可以把事情解释得很清楚     │ │
│ └─────────────────────────────────┘ │
│ ┌─────────────────────────────────┐ │
│ │ 00:28  铺垫                    │ │
│ │ 让我这种很门外汉的人也知道他...  │ │
│ └─────────────────────────────────┘ │
│ ┌──────────────────────┐           │
│ │ 00:32  方法           │  ← 鼠标悬停态：灰色背景           │
│ │ 他会告诉你哪些是你需要的│  💬 "可以直接定位到对应的时间线" │
│ └──────────────────────┘           │
│                                     │
│ ████████░░░░░░░  [滚动条]           │
└─────────────────────────────────────┘
```

**每张卡片包含：**
1. **时间戳**（精确到秒）：00:25, 00:28, 00:32...
2. **说话人/段落标签**：方法、铺垫、植草、review 等
3. **转写文本**：原始对白内容

### 3.2 交互状态

| 状态 | 视觉效果 | 触发条件 |
|------|---------|---------|
| **默认态** | 白色背景 | 非当前播放位置 |
| **播放高亮态** | 蓝色背景 | 视频播放到对应时间戳 |
| **悬停态** | 灰色背景 + tooltip「可以直接定位到对应的时间线」 | 鼠标悬停 |
| **选中/点击后** | 视频播放器跳转到该时间戳 | 点击卡片 |

### 3.3 对我们 MVP 的启示

✅ **可以直接参考的模式：**
1. 对白转写以**卡片列表**形式呈现，每条带时间戳
2. **播放时自动高亮**当前卡片 = 视觉锚点
3. **点击卡片跳转视频** = 选取 → 定位
4. 右侧 AI 分析面板也支持点击时间戳范围跳转

🔧 **可以增强的点（百万工作室没做但我们可以做）：**
1. **多选标记**：选中多张对白卡片 → 批量标记 → 导出标记片段
2. **标记分类**：不同颜色的标记（好钩子🟢、金句🟡、转折点🔴）
3. **标记 → 知识卡自动生成**：选中对话片段 → 一键提取为知识卡
4. **标记时间轴总览**：在视频时间线上显示所有标记位置

## 四、知识卡片系统

### 4.1 卡片结构

在「知识库」中，每张知识卡是**高度结构化的数据单元**：

```
┌──────────────────────────────────────────┐
│ 🤖 AI工具与工作流 · 偏差-长期有效           │
│                                          │
│ 📌 GPT作为CTO角色：审查技术方案...          │
│                                          │
│ 📖 定义：将GPT用作CTO角色。审查Codex...    │
│ 💡 核心要点：非技术开发者缺乏技术审查能力... │
│ 🔧 怎么用：让Codex生成技术方案→GPT审查→... │
│ 🎯 适用场景：非技术背景的AI使用者...        │
│ 📎 来源引用："我发觉情侣还好有一个GPT..."   │
│                                          │
│ 🏷️ 标签：AI编程 工作流 建议 规划            │
│ ⚠️ 时间敏感  ⏱️ 长期有效  🔰 新手          │
│                                          │
│ [▶ 跳到视频片段]  ← 与源视频位置联动！      │
└──────────────────────────────────────────┘
```

### 4.2 知识卡与视频的关联

每张知识卡底部都有 **「▶ 跳到视频片段」** 按钮，点击后：
- 视频播放器跳转到该知识提取的源时间戳
- 对白转写卡片同步高亮到对应位置

**这是「标记 → 定位」的另一种实现：**
- 知识卡 = 已保存的「标记」
- 跳转按钮 = 从标记定位到视频位置

### 4.3 知识卡的分类体系

```
知识库（47张）
├── AI工具与工作流（29张）
│   ├── 工具筛选
│   ├── 工作流
│   └── 提示词
├── 内容表达与脚本结构（4张）
├── 案例拆解方法（4张）
├── 平台规则与趋势判断（8张）
└── 个人经验/观点沉淀（1张）
```

**筛选维度：**
- 搜索：定义、步骤、适用场景、来源片段、标签
- 过滤器：全部难度（新手/进阶/高级）、全部新旧风险
- 标签：时间敏感、关联分类

## 五、完整创作工作流

### 5.1 输入：导入素材

**两种模式：**
- **批量链接**：URL + 标题 + 平台 + 创作者 + 好在哪 + 标签
- **本地文件**：mp4/mov/m4v + srt/vtt/txt 字幕

交互细节：
- 平台字段「可留空，自动读取」→ 智能识别
- 标签提示「AI工具、口播、反常识」→ 引导用户
- 底部「批量任务」进度面板 → 异步任务状态透明

### 5.2 分析：脚本详细拆解

**四栏布局：**
```
┌──────────┬──────────────┬─────────────────────────┐
│ 视频播放器│ AI分析报告    │ 自动生成的「知识卡式」摘要  │
│          │ Tabs:        │                         │
│          │ · 内容总览    │ 01:10-01:24 evidence    │
│ 原文时间轴│ · 结构拆解    │   优点/缺点/改写          │
│ (对白卡片) │ · 复刻模板    │                         │
│          │ · 新脚本方向   │ 01:24-02:13 method      │
│          │ · 三项评分    │   对比验证+个人体验        │
│          │ · 高级       │                         │
└──────────┴──────────────┴─────────────────────────┘
```

**复刻模板（结构化公式提取）：**
- 开头公式：问题+推荐 →「有没有【痛点】的【工具类型】？我推荐【工具名称】」
- 正文公式：对比论证+个人体验 →「以前我用【旧工具】，但最近转向了【新工具】...」
- 案例公式：工作流步骤+个人感受 →「我有个小技巧：千万不要【错误做法】...」

### 5.3 输出：一键成稿

**输入选择面板：**
- 📝 选题选择（可收藏、可搜索、按受众/价值筛选）
- 🃏 知识卡选择（最多8张、checkbox多选、搜索/筛选）
- 📄 参考脚本/复刻方向
- ⚙️ 生成参数：
  - 平台：小红书/抖音/视频号/B站 → **自动调整时长**
  - 时长：90s/180s 可调
  - 风格：真诚 / 口播
  - □ 自动带入我的账号设置

**生成结果面板：**
```
┌─────────────────────────────────┐
│ [复制脚本]                       │
│                                 │
│ 📌 标题文案：「突然被cue...」     │
│ 🎯 封面文案：「3个公式告别...」   │
│                                 │
│ 🪝 Hook：                       │
│   有没有这种时刻？开会突然被点名...│
│                                 │
│ 📝 完整脚本：                    │
│   第一个公式：观点-案例-观点       │
│   第二个公式：问题-解决方案        │
│   第三个公式：现在-未来           │
│   结尾 - 10s                    │
│                                 │
│ 🎬 拍摄建议                      │
│ ✂️ 剪辑建议                      │
└─────────────────────────────────┘
```

### 5.4 个性化：我的账号

**账号定位：**
- 账号名、一句话定位
- 核心受众、受众痛点、价值主张
- 内容支柱、内容边界
- 风格方向、平台注意事项

**写作规则（多维度约束）：**
- 开头规则 / 结构规则 / 描述规则
- 禁用句式（如「不是…而是…」）
- 平台差异规则
- 可用规则

**自动带入机制**：一键成稿时可勾选「自动带入我的账号」，确保生成内容符合账号人设。

## 九、结论

百万工作室的核心交互模式——**「对白卡片 ↔ 视频时间轴」双向联动**——正是 Daryl 想要参考的功能。

它的实现方式：
1. 对白以卡片形式呈现，每条带时间戳
2. **播放高亮** + **点击跳转** = 双向绑定
3. 知识卡 = 结构化标记，底部按钮跳回视频源位置
4. 缺失的是**显式的「标记」操作**——这是我们的差异化机会

**建议我们做的差异化：**
- 对白卡片上增加**显式标记按钮/快捷键**
- 支持**多选 + 批量标记**
- 标记结果在**视频时间轴上可视化**
- 标记 → 知识卡的一键转换
- 标记的**颜色分类**（钩子/金句/转折/数据/情感点）
