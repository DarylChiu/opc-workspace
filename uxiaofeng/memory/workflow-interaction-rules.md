# Workflow 交互规则 v1.0

> 适用范围：全部 4 个 Agent（忧郁小猫、吹点小风、算点小账、恨点小己）
> 最后更新：2026-07-05

---

## 一、核心原则

### 1. 单次回复原则（ONE REPLY ONLY）

收到 Workflow 变更通知后，**必须且只能回复一次**。

- ✅ 先完整分析所有变更，再输出最终结论
- ❌ 秒回第一反应，再补修正消息
- ❌ 分多段回复（除非 Daryl 追问）

### 2. 推理优先原则（REASON FIRST）

在看到变更通知到输出回复之间，**必须执行完整推理**：

1. 读取 Workflow 完整数据（节点、连线、预估时间、依赖关系）
2. 逐节点分析变更内容
3. 检查上下游影响和工时合理性
4. 形成结论后，一次性输出

### 3. 无歧义原则（NO AMBIGUITY）

- 如果变更中有不明确的地方 → 在单次回复中列出所有疑问，等待 Daryl 确认
- 不要先回一个模糊确认、再回一个追问
- 计算类节点必须先执行计算，再给结论，不要给「可能」「大概」

## 二、收到 Workflow 变更的标准流程

```
┌─────────────────────────────────────────┐
│ 1. 收到通知                              │
│    识别变更类型（节点修改/新增/删除/连线）  │
├─────────────────────────────────────────┤
│ 2. 读取完整 Workflow 数据                 │
│    GET /api/workflow/:id 或读取本地JSON   │
├─────────────────────────────────────────┤
│ 3. 逐节点分析                             │
│    - 变更了什么？                         │
│    - 我的节点预估是否合理？                │
│    - 上下游依赖是否受影响？                │
│    - 总工时是否在可接受范围？              │
├─────────────────────────────────────────┤
│ 4. 执行验证计算（如有）                    │
│    涉及数值/工时的必须先算完              │
├─────────────────────────────────────────┤
│ 5. 一次性输出结论                         │
│    - 确认/有异议                         │
│    - 如有疑问，在本次回复中全部列出        │
│    - 不要分段、不要追加                    │
└─────────────────────────────────────────┘
```

## 三、回复格式规范

```
## [确认/有异议] Workflow [名称] v[版本] 变更分析

### 📋 变更概览
- 节点X: [变更内容]
- 节点Y: [变更内容]

### ⏱ 工时评估
| 节点 | 预估 | 评估 | 备注 |
|-----|------|-----|------|
| ... | ... | ✅/⚠ | ... |

### 🔗 依赖检查
- [上下游影响分析]

### ✅ 结论
[一次性结论，无歧义]
```

---

## 四、禁止行为

| ❌ 禁止 | ✅ 正确做法 |
|---------|-----------|
| 看到通知秒回「收到」再补分析 | 分析完再一次回 |
| 第一条回「我觉得不对」第二条回「算了一下确实不对」 | 先算完，然后给出有依据的结论 |
| 不读完整 Workflow 就回复 | 先读数据，再分析 |
| 回复里包含「可能」「大概」「应该」等模糊词（涉及计算时） | 先算清楚，再给确切结论 |
| 变更影响我负责的节点时直接说「没问题」不解释 | 说明为什么不影响 |

---

## 五、特殊情况

### 变更涉及多 Agent 协调
- 如果变更影响其他 Agent 的节点 → 在回复中标注，由 Daryl 决定是否需要通知
- 不要自作主张替其他 Agent 回复

### Daryl 的工时缩短
- Daryl 缩短预估工时 → 评估可行性，不能实现就说不能，不要强行答应
- 可以提议替代方案

### 无影响变更
- 即使无影响，也需要回复确认（一次），说明已检查并确认无影响
