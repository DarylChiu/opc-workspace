### IELTS Tutor v0.9.1 — LLM延迟优化
- **根因发现**: `deepseek-v4-flash` 默认开启 thinking 模式（chain-of-thought），每次请求都产生大量 reasoning tokens 吃掉配额+增加延迟+导致空回复
- **修复**: 所有 API 调用添加 `"thinking": {"type": "disabled"}`，model 名从 `deepseek-chat` 改为 `deepseek-v4-flash`
- **附带优化**: 历史缩减到 4 轮 (8 条消息)、max_tokens 固定 200
- **Debug 面板**: 新增 `/debug/turns/{sid}` HTTP 轮询端点 + 前端 HTTP fallback（解决平板 WebSocket 不通）、「本轮详情」默认折叠
- 预期 LLM 延迟从 4000ms 降到 1000-2000ms
### IELTS Tutor v0.9.1 — 成本核算修复 + 语音截断修复
- **成本 Bug 发现**: TTS/STT 用了 Google Cloud 定价公式计算成本，但实际用的是本地 Kokoro+whisper（免费），$1.08 账单中 $1.07 是假的
- **修复**: 成本端点重写，只计算 DeepSeek V4 Flash ($0.14/$0.28 per 1M)，实际成本接近 $0

- **语音截断修复**: STT 过滤器阈值从 20 字符降到 5 字符，"yes"/"no" 等正常短回复不再被过滤；音频缓冲区从 30s 扩到 60s
- **ds_stream**: 从 streaming 响应的最后一个 chunk 捕获 usage token 计数
