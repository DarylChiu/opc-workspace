Daryl 昨天（7/14 晚）做了一轮管线1自由对话测试，反馈三个问题，我当晚诊断如下：
- **问题1 长句断句**: 根因 = audio buffer 30s 硬上限（`960000 bytes` auto-flush），长难句被物理切成两截
- **问题2 TTS 发音错误**: 根因 = Kokoro-82M 的 G2P（misaki/espeak-ng）音素转换精度不足（日志 `words count mismatch 100%`）。→ 昨晚已执行方案A：翻转 Piper high 为主引擎、Kokoro fallback（已部署）
- **问题3 评估不显示**: 根因 = 评估实际成功（`overall=5.5`，HTML 报告已生成），但 `ws.send_json` 时 WebSocket 已断开，评分发不出去

### 今天（7/15 上午）修复问题1 + 问题3
Daryl 追问这两个是否解决——之前只做了问题2（TTS翻转），问题1/3 未落地。今天补上：

**问题1 长句断句** (`backend/server.py`)
- 30s 硬切 → 双阈值：软上限 60s（`AUDIO_BUF_SOFT_SECONDS`）+ 硬上限 90s（`AUDIO_BUF_HARD_SECONDS`）
- 软上限触发后**等说话停顿**（>0.6s 静音，`AUDIO_BUF_FLUSH_SILENCE`）才 flush，不在句子中间硬切
- 硬上限才无条件强切（防内存失控）
- 均可用环境变量调

**问题3 评估不显示** (`backend/server.py` + `frontend/index.html`)
- 后端：评估结果同时缓存到 `session["last_score"]`，新增 HTTP 兜底路由 `GET /api/session/{sid}/score`（WS断开也能取）
- 前端：WS `onclose` 或 15s 未返回评分 → 自动切 HTTP 轮询 `pollScore()`（每3s，最多60s），拿到就渲染；超时提示"可在历史报告查看"
