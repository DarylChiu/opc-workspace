# 2026-07-11 日记

## Daryl 对话 — Sentinel 事故复盘 + Loop Engineering 改进讨论

### 问题
Daryl 反馈 Sentinel 看门狗系统存在严重问题：
1. **$20 OpenRouter API 成本损失** — 重启重连重放 + defer 死锁期空转 + Cron 多 Agent 响应叠加
2. **Gateway 持续重启** — 7/7 当天 4+ 次连锁重启，Agent 开发效率下降
3. 根本原因：Sentinel v1 Plugin 的 `before_tool_call` hook 拦截全部工具（含只读），依赖 Gateway 通信做审批，审计日志写入触发 config hot-reload → restart 正反馈震荡

### 讨论过程
1. **我最初提议**：放弃 Plugin，转向 Agent 内置判断 + 声明式策略
2. **Daryl 反驳**：Agent 自觉不可靠，内置判断 = 回到原点，达不到 Loop Engineering 效果
3. **我认同**：这是委托-代理问题，Agent 天然会跳过合规检查，必须外部强制
4. **我还是坚持 Gateway 级 hook 机制**，但要重建 Sentinel v2 — 关键改动：
   - 只拦截 write/edit/exec（放过读操作）
   - 全部内联判断（零 Gateway 通信）
   - P0 用 block: true（不依赖审批）
   - 不写 audit log（根除重载触发）
   - disable 即时生效，零开销
5. **查阅 OpenClaw 原生工具策略**：tools.profile/allow/deny 只能整刀切工具，无法做路径级判断 → before_tool_call hook 是唯一可行机制

### 待决策
Daryl 认为基建问题需要更强模型才能从根本上解决，明天继续讨论。
