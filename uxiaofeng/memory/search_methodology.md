# 搜索方法论 · Search Methodology

> 适用: 全体Agent | 更新: 2026-07-17 | 引擎: Brave API(主力) + Bing(兜底)

## 一、关键词构造策略

### 1.1 多维度拆解
不要直接扔原始query。拆解为3-5个维度的关键词组合：
- **主体名**: 作品名/人物/事件（中英文各一份）
- **功能标签**: 分析/拉片/拆解/影评/review/breakdown
- **情感/类型标签**: 经典/催泪/反转/高能/经典桥段
- **技术特征**: 运镜/剪辑/台词/表演/镜头语言
- **平台限定**: site:zhihu.com site:douban.com site:cinephilia.net

**示例**: 查「电梯对峙桥段」
```
❌ 搜: 电梯对峙
✅ 搜: "电梯" "对峙" 电影桥段 分析 site:zhihu.com
✅ 搜: elevator confrontation scene film analysis
```

### 1.2 中英文双搜
每个查询同时发中英文各一次，结果合并去重。
- 中文 → Brave API（language=zh）
- 英文 → Brave API（language=en）
- 兜底 → Bing（两个语言都搜）

## 二、首搜失败降级策略

### 降级阶梯
| 轮次 | 动作 | 说明 |
|---|---|---|
| 第1轮 | 完整关键词（主体+功能+情感+技术） | 最精确 |
| 第2轮 | 去修饰词，保留核心名词 | 降维 |
| 第3轮 | 切换语言（中→英 或 英→中） | 跨语言 |
| 第4轮 | 只用主体名+最宽泛功能词 | 最大召回 |
| 第5轮 | 换引擎（Brave→Bing→Bing+site:） | 兜底 |

### 降维规则
- 去掉形容词（精彩的/感人的/经典的）
- 去掉副词（非常/极其）
- 保留：作品名、核心名词、功能动词

## 三、结果过滤规则

### 🚫 黑名单（自动过滤）
| 类型 | 域名/关键词 | 原因 |
|---|---|---|
| 百科词条 | baike, wikipedia(非分析类), MBA智库 | 定义性内容无分析价值 |
| 词典 | cidian, dict, 词典, 翻译 | 无意义 |
| 商业推广 | 广告, 推广, 购买, taobao, jd.com | 非内容 |
| 低质聚合 | 360doc, 百文网, 瑞文网 | 洗稿源头 |
| AI生成站 | 无明显署名, 无评论区的短内容站 | 无信源价值 |

### ✅ 白名单提权（加权排序）
| 类型 | 域名 | 权重 |
|---|---|---|
| 专业影评 | zhihu.com, douban.com, cinephilia.net | +3 |
| 学术/半学术 | cnki.net, 学术期刊 | +3 |
| 高质量自媒体 | 有署名、有分析的公众号/博客 | +2 |
| 视频平台 | bilibili.com(长视频), youtube.com(>10min) | +2 |
| 新闻/媒体 | 正规媒体娱乐版 | +1 |

### 相关性评分（快速筛）
对每条结果快速打分（0-5）：
- 标题包含核心主体名：+2
- 标题包含功能词（分析/拉片/拆解/review）：+1
- URL在白名单中：+1
- 发布时间<2年：+1
- 摘要 > 100字且有实质内容：+1

**阈值：≥3分保留，<3分丢弃**

## 四、搜索执行规范

### 每次搜索
1. 构造2-3组不同维度的关键词
2. 每组中英文各搜一次（共4-6次搜索）
3. 每次取前10条
4. 全部结果合并去重后打分
5. 取top 10高分结果

### 结果不足时
- 总分>2的结果 < 3条 → 启动降级策略
- 3轮降级后仍不足 → 标记为「信息不足」并报告

### 引擎优先级
```
默认: Brave API → Bing
Brave额度用尽 → Bing → Bing + site:过滤
中文专项: Brave API(zh) → sogou wechat → Bing(zh)
```
