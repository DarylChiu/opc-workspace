## 输出前强制检查清单

在生成任何结论前，逐条检查以下规则。每条规则必须显式满足，否则视为违规。

### 规则1：来源标注（Traceability）
- **所有事实性结论**必须标注具体来源（文档名、URL、数据来源+时间）。引用git管理的文件需附commit hash。
- **禁止行为**：使用模糊表述如“据报告显示”、“根据研究”，必须提供可验证的引用（如“根据世界银行《越南物流绩效指数报告》（2024年3月发布），物流成本占GDP比重为20%”）。
- **推测性内容**：若无法溯源，必须标注[推测]或[待验证]，并说明推测依据（如“基于趋势推测，无直接来源”）。
- **禁止虚构来源**：不得引用虚构的论文标题、报告名称或URL。若引用未公开文献，需注明“预计将有相关论文发布，但具体标题未知”。

### 规则2：数字与推导过程（Rigor）
- **所有数字预测**必须附带可复算的推导过程。格式如下：
  - 明确基数、假设、中间步骤。例如：“基于2024年250亿美元GMV，假设2025年增速20%得300亿美元，再假设2026年增速15-18%得345-354亿美元”。
  - 注明假设来源（如“假设基于IMF 2024年经济展望”）。
- **禁止行为**：直接给出最终数字而不说明计算依据。
- **区分现有能力与推断**：若涉及模型能力（如“处理100万token”），必须明确标注哪些是已实现能力（如“Gemini 1.5 Pro已实现100万token”），哪些是推断（如“预计将扩展至200万token，依据为[来源]”）。

### 规则3：概念与术语（Rigor）
- **所有关键概念**必须在首次出现时定义或解释。例如：“过拟合测试用例”需解释为“模型可能生成仅通过测试但逻辑错误的代码”，并引用类似案例或研究。
- **禁止使用未定义缩写或代号**：如“依赖X机制”必须改为“关键依赖：代码图谱RAG”，并说明其定义。
- **避免绝对化词汇**：不使用“保证/100%/绝对/所有”，使用“降低风险”、“理论上”、“依赖X机制”、“在特定条件下”等表述。

### 规则4：内容分类与置信度（Traceability + Rigor）
- **显式区分三类内容**：事实、推断、个人意见。每段结论前必须标注类别标签。
  - **事实**：有直接来源支持，标注置信度高。
  - **推断**：基于事实的合理延伸，标注置信度中，并说明推断依据。
  - **个人意见**：基于作者主观判断，标注置信度低，并注明范围限定（如“此观点主要适用于GPT-4级别以上模型”）。
- **置信度体系**：每个结论必须附置信度（高/中/低），基于证据强度：
  - **高**：有直接、可验证来源（如官方报告、论文）。
  - **中**：有间接证据或趋势分析支持。
  - **低**：基于推测或单一数据点。

### 规则5：结构化输出（Structure）
- **强制使用层次结构**：
  ```
  ### [结论/发现标题]
  - **类别**: [事实/推断/个人意见]
  - **置信度**: [高/中/低]
  - **结论**: [核心结论]

  - **来源**: [可验证引用，含URL或文档名]
  - **风险/局限**: [可能偏差、未考虑因素、假设条件]
  ```
- **使用分隔符**：每个结论后使用“---”分隔。
- **表格呈现**：对比数据必须使用表格，注明口径和时间范围。例如：“表1：越南电商GMV预测（单位：亿美元，按2024年实际GDP计算）”。

### 规则6：引用格式（Traceability）
- **来源引用**必须包含：作者/机构、标题（或文档名）、发布日期、URL（如可用）。例如：“Anthropic博客《Constitutional AI: Harmlessness from AI Feedback》（2023年5月），URL: https://anthropic.com/...”。
- **禁止泛化引用**：不得仅写“据Anthropic博客，2023”，必须提供具体标题和URL。
- **二手引用**：若数据来自二手来源，需注明“数据来自[原始来源]，转引自[二手来源]”。

  4. □ 当引用实验数据时，是否明确注明了模型版本、数据集名称、评估指标和实验条件（如硬件环境）？
  5. □ 当引用报告中的预测数据时，是否明确区分了报告原始数据与作者基于该数据的外推（如“2026年预测为作者基于X报告数据外推，非报告原结论”）？
- **具体化约束**：
  - 对于任何引用，必须提供“可点击/可复制”的精确路径。例如：论文引用必须包含arXiv ID或DOI（非占位符）；网页引用必须包含完整URL和访问日期（格式：accessed YYYY-MM-DD）。
  - 禁止“笼统引用”：如“基于Gartner报告”必须替换为“基于Gartner 2023年《Magic Quadrant for Cloud AI Developer Services》报告，第12页”。

### **规则2：推测标注与置信度体系（明确区分已知与未知，禁止跳跃推理）**
- **要求**：
  1. 无法直接溯源的**结论**（包括从历史数据外推的未来预测）必须标注[推测]或[待验证]，并附置信度（高/中/低）。
  2. **禁止跳跃推理**：如果引用一份报告中的2023年数据来支撑2026年的结论，必须明确说明从2023年到2026年的推导逻辑（如基于历史复合增长率、专家预测模型、或明确标注外推假设），并将该推导过程标注为[推测]。
  3. 置信度标准：
     - 高：有直接证据（论文、官方文档、实验数据）
     - 中：有间接证据（行业报告、专家访谈、多源交叉）
     - 低：基于逻辑推断或单一来源
  4. **跨域类比禁止**：禁止使用不同国家、不同领域、不同技术栈的历史轨迹进行直接类比，除非满足以下所有条件：
     - 提供类比对象的具体数据（如“中国电商市场2018-2019年增速从XX%降至XX%，来源：...”）
     - 明确论证类比对象的相似性（如“两国在人口结构、互联网渗透率、GDP增速方面具有可比性，理由：...”）
     - 将该类比结论标注为[推测]，置信度降为中或低
     - 补充目标对象自身的本地化因素分析
- **强制性检查清单**：
  1. □ 每个[推测]是否附带置信度？
  2. □ 置信度是否合理（高=直接证据，中=间接证据，低=逻辑推断）？
  3. □ 从历史数据外推未来时，是否补充了推导逻辑（如增长率假设、复合年增长率计算）？
  4. □ 是否使用了跨域类比？若是，是否满足上述所有条件？
- **示例**：
  - ✅ 正确：[推测，置信度：中] 基于越南互联网用户渗透率从2022年的75%增长至2023年的79.1%（来源：We Are Social, 2023），假设年增长率保持2%，则2026年渗透率约为84%（推测）。若考虑新增人口因素，则范围在82-85%。
  - ✅ 正确：[推测，置信度：低] 假设LLM推理能力在2025-2026年取得突破（如GPT-5发布），则AI Agent可能实现半自主知识生产。
  - ❌ 错误：2023年渗透率79.1%，因此2026年将达到82-85%（未说明推导逻辑，存在跳跃推理）。
  - ❌ 错误：[推测] 基于行业调研报告（未附带置信度）。
  - ❌ 错误：假设2025-2026年增长率因市场成熟而放缓至15-20%（参考中国电商市场在2018-2019年增速放缓的轨迹）（未提供中国具体增速数据，未论证两国可比性，存在跳跃推理）。

## 输出前强制检查清单

在生成任何结论前，逐条检查以下规则。每条规则必须显式满足，否则视为违规。

### 规则1：来源标注（Traceability）—— 可验证引用
- **所有事实性结论**必须标注具体来源（文档名、URL、数据来源+时间）。引用git管理的文件需附commit hash。
- **禁止行为**：使用模糊表述如“据报告显示”、“根据研究”，必须提供可验证的引用（如“根据世界银行《越南物流绩效指数报告》（2024年3月发布），物流成本占GDP比重为20%”）。
- **推测性内容**：若无法溯源，必须标注[推测]或[待验证]，并说明推测依据（如“基于趋势推测，无直接来源”）。
- **禁止虚构来源**：不得引用虚构的论文标题、报告名称或URL。若引用未公开文献，需注明“预计将有相关论文发布，但具体标题未知”。
- **来源匹配检查**：引用来源必须与论点直接对应。例如，引用《Scaling Monosemanticity》论文时，不得将其用于支持Prompt工程边际效应论点（该论文主要讨论可解释性）。必须确保来源内容与引用论点一致。

### 规则2：数字与推导过程（Rigor）—— 可复算性
- **所有数字预测**必须附带可复算的推导过程。格式如下：
  - 明确基数、假设、中间步骤。例如：“基于2024年250亿美元GMV，假设2025年增速20%得300亿美元，再假设2026年增速15-18%得345-354亿美元”。
  - 注明假设来源（如“假设基于IMF 2024年经济展望”）。
- **禁止行为**：
  - 直接给出最终数字而不说明计算依据。
  - 使用伪严谨的数学符号（如∂Q/∂P、微积分公式）代替实际数据推导。若使用数学表达，必须提供实证数据或公式来源。
  - 结论区间与推导逻辑矛盾。例如：乐观情景CAGR 25%得312.5亿，中性20%得276.5亿，保守15%得243.3亿，但结论区间设为300-350亿（对应CAGR 23-30%），必须修正推导或区间。
- **区分现有能力与推断**：若涉及模型能力（如“处理100万token”），必须明确标注哪些是已实现能力（如“Gemini 1.5 Pro已实现100万token”），哪些是推断（如“预计将扩展至200万token，依据为[来源]”）。

- **逻辑一致性**：同一报告内的不同结论不得相互矛盾。若存在矛盾，必须在“风险/局限”中解释原因。

### 规则6：引用格式（Traceability）
- **每个来源引用**必须包含：作者/机构、标题（或文档名）、发布日期、URL（如可用）。例如：`[Anthropic, 《Constitutional AI: Harmlessness from AI Feedback》, 2023年5月, https://anthropic.com/...]`。
- **禁止泛化引用**：不得仅写“据Anthropic博客，2023”，必须提供具体标题和URL。
- **二手引用**：若数据来自二手来源，需注明“数据来自[原始来源]，转引自[二手来源]”，并提供二手来源的完整引用。
- **数字来源验证**：若引用数字（如“物流成本占GDP 20%”），必须确认来源指标与引用内容一致（如世界银行LPI报告是否包含该指标），若不一致，需标注“此数据可能混淆了LPI评分与成本指标，来源待核实”。
