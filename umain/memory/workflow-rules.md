# 工作流规范 (Daryl确认 2026-06-06)

## 1. 决策授权矩阵

| 级别 | 响应时限 | 场景 | 处理方式 |
|------|---------|------|---------|
| **P0 紧急** | 30分钟 | 系统崩溃、数据丢失 | 执行应急方案，事后汇报 |
| **P1 重要** | 2小时 | 架构方向、核心技术选型 | 必须请示Daryl，提供2-3方案+推荐；超时→执行推荐方案 |
| **P2 正常** | 6小时 | 实现细节、代码优化 | 可自主决策；超时→按最佳理解执行 |
| **P3 参考** | 24小时 | 代码风格、文档完善 | 不等待直接做 |

**核心原则**：技术实现归Agent，架构方向归Daryl
**超时机制**：绝不停滞，超时执行预设低风险方案
**禁止区**：金钱交易、核心安全、非授权隐私 → 绝对不可自主决策

## 2. 里程碑驱动工作流

| 规模 | 预估工时 | 里程碑数 | 缓冲 |
|------|---------|---------|------|
| 小型 | <40h | 2-3个 | 15% |
| 中型 | 40-120h | 3-4个 | 20% |
| 大型 | >120h | 4-6个 | 25% |

**汇报节点**: 启动 / 30% / 60% / 90% / 100%
**每个里程碑**: 边界、交付物、验收标准、决策权限

## 3. 汇报格式

```
## [任务名] 里程碑X 进度汇报 (XX%)
调用模型: xxx

### ✅ 已完成
### 🔄 进行中
### ⚠️ 问题/风险
### ❓ 决策需求
### 📝 自主决策记录
### 🔜 下一步
```

- 仅里程碑节点汇报，不频繁打断Daryl
- 紧急 → 加【紧急】标签
- 重要 → 加【重要】标签
- 自主决策每次记录理由，里程碑汇报时一并提交
