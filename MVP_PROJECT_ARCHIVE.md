- **解决方案**: 清理进程 + 修复SSL + 完全内联HTML

- **时间**: 2026年4月7日

### 架构演进历程

#### 方案对比

| 方案 | 描述 | 优点 | 缺点 | 选择原因 |
|------|------|------|------|----------|
| **方案A** | HTML内嵌到FastAPI后端 | 部署简单，单端口，稳定性高 | 前端开发灵活性受限 | MVP阶段首选 |
| **方案B** | 前后端分离双端口架构 | 前后端独立开发，技术栈灵活 | ngrok代理复杂，请求处理困难 | 不选择 |

#### 架构演进图
```
初始设计 → 双端口架构 → ngrok困境 → 版本冲突 → 进程残留 → 
简化设计 → 单端口内嵌 → SSL修复 → 完全内联 → 功能验证
```

### 技术问题清单

#### 问题分类表

| 问题类别 | 具体问题 | 根本原因 | 解决方案 | 严重程度 |
|---------|---------|---------|---------|---------|
| **部署架构** | ngrok无法同时代理前后端 | 多端口分离设计 | 方案A：HTML内嵌到单一服务 | 🔴 高 |
| **版本控制** | 用户看到繁体中文旧版本 | 进程残留导致版本冲突 | 强制终止旧进程，验证部署版本 | 🔴 高 |
| **资源依赖** | styles.css、app.js 404错误 | HTML仍有外部资源引用 | 完全内联HTML，无任何外部引用 | 🟡 中 |
| **网络连接** | SSL证书验证失败 | macOS Python证书链不完整 | 安装证书包 + 代码端禁用验证 | 🟡 中 |
| **交互设计** | 只有TTS输出，无用户输入处理 | MVP原型设计不完整 | 建立完整交互流程 | 🔴 高 |

### 服务器核心代码

#### 版本标志
```python
"version": "1.1.0-full-interaction",
"tts_configured": bool(GOOGLE_TTS_API_KEY),
"message": "Bryson语音交互MVP - 完整交互架构"
```

#### 关键端点
1. `GET /` - 主页面（完全内联HTML）
2. `GET /status` - 服务器状态检查
3. `POST /api/tts` - TTS语音合成
4. `POST /api/process_recording` - 处理录音（预留STT）

### HTML内嵌实现

#### 三个"无"原则
1. **无`<link>`标签**: 所有CSS在`<style>`标签内
2. **无外部`<script src="">`**: 所有JavaScript在`<script>`标签内
3. **无相对路径资源请求**: 避免浏览器请求不存在的文件

#### 代码示例
```python
html_content = f"""
<!DOCTYPE html>
<html>
<head>
    <meta charset="UTF-8">
    <title>🎤 Bryson 实时语音练习室</title>
    <style>
        /* 所有CSS内联在此 */
        body {{ font-family: -apple-system, BlinkMacSystemFont, sans-serif; }}
    </style>
</head>
<body>
    <h1>🎤 Bryson 实时语音练习室</h1>
    <script>
        // 所有JavaScript内联在此
        let interactionCount = 0;
    </script>
</body>
</html>
"""
```

### STT预留接口设计

#### 设计理念
- **接口标准化**: 统一处理录音数据的接口
- **模块化分离**: STT逻辑可独立替换
- **模拟数据支持**: 开发阶段使用模拟用户输入

#### 实现代码
```python
@app.post("/api/process_recording")
async def process_recording(request: Request):
    # 模拟用户输入（未来STT替换这里）
    user_input = request_data.get("user_input", "User completed a recording practice.")
    
    # 智能反馈生成逻辑
    feedback_text = generate_feedback_response(interaction_count, user_input)
    
    # TTS合成返回
    audio_content = synthesize_speech(feedback_text)
    
    return {
        "feedback_audio": audio_content,
        "interaction_number": count,
        "user_input_placeholder": user_input
    }
```
