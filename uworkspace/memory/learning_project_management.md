# Learning: 项目管理经验

## 来源: 双系统协同开发项目 (2026-04-06 ~ 2026-04-16)

### 核心教训

1. **决策依赖循环风险**
   - "汇报→等待→停滞"循环是最大项目杀手
   - 建立默认决策机制：等待超5天自动执行预设方案

2. **同步等待模式风险**
   - 识别同步决策依赖导致的停滞，改用异步工作

3. **技术依赖风险管理** (来自48小时流程重构项目)
   - 技术配置问题可能阻塞整个验收
   - 对策：提前验证所有依赖项，不要在deadline当天发现

4. **时间管理优化**
   - 48小时项目实际用了更长时间
   - 对策：项目计划中明确里程碑，超时就escalate

5. **归档管理**
   - 区分活跃项目与归档项目汇报机制
   - 建立项目超时自动归档规则
   - 归档后不再在进度报告中提及
