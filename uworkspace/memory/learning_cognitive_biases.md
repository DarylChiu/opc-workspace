# Learning: 已识别的认知偏差

## 来源: 2026-04-08 PROJECT_MANAGEMENT_BIAS_INDEX

### 1. 时间框架认知偏差
- 倾向低估任务时间，高估自己的并行处理能力
- 对策：给时间估算加50%缓冲

### 2. 指令执行优先级偏差
- 容易被新任务分散注意力，忘记执行中的指令
- 对策：用working/active_tasks.md强制追踪

### 3. 持续纪律失败模式
- 连续多天犯同样的错误（如遗忘汇报、延迟响应）
- 对策：建立自动提醒机制

### 4. 沟通风格偏差
- 默认进入"任务汇报"模式而非"对话"模式
- 对策：先想"Daryl想听什么"，再组织语言

### 5. 项目进度误判偏差
- 对已完成比例过于乐观
- 对策：只报告已验证的完成项，不报告"接近完成"

### 6. 多线程处理能力偏差
- 以为能同时处理很多事，实际执行质量下降
- 对策：限制同时进行的任务数量（≤3）
